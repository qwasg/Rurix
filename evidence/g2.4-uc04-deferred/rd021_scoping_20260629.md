## 1. 目的

按本任务"第 0 步必做:RD-021 决策树"要求,先判定 UC-04 deferred 渲染器的 lighting pass 是否可以在**不触纹理采样语义本体**的前提下由 Rurix source 一次做到 device 绿。只有第一分支成立,后续 device 绿才允许一次做到底;否则**停手升档**,产出本 scoping 交 agent,并做满"不采样"的可达面。

判定结论:**第一分支不成立 → 停手分支**。本任务不出 device 绿,G-G2-4 维持 blocked-honest 交接 agent。

## 2. 复核事实(均经本机实读,附文件:行)

### 2.1 图形=B body 降级能力边界 = RXS-0171 白名单,无任何采样/取数路径

`src/rurixc/src/dxil_spirv.rs` `BodyLowerer`(`emit_spirv_body` 的 body 降级器)白名单逐字核对:

- 支持的 rvalue:`Use` / f32·i32·u32 `Const` / 标量·向量 `BinaryOp(加减乘除)` / 输出 I/O 聚合返回值机械分解([src/rurixc/src/dxil_spirv.rs:696-708](../../src/rurixc/src/dxil_spirv.rs))。
- 输入 I/O 字段 → `OpLoad`([src/rurixc/src/dxil_spirv.rs:974-982](../../src/rurixc/src/dxil_spirv.rs));输出 I/O 元素 → `OpStore`([src/rurixc/src/dxil_spirv.rs:1017-1035](../../src/rurixc/src/dxil_spirv.rs))。
- **资源/纹理/采样访问 → `RX6013` strict-only 拒**(spec/dxil_backend.md RXS-0171 L4;`BodyLowerer::lower_rvalue_any` 对 `Rvalue::Use` of 资源 place 无路径,白名单外 rvalue 一律 `DxilError::Unmappable` → `DxilBError::Spirv` → RX6013)。
- SPIR-V opcode 常量表([src/rurixc/src/dxil_spirv.rs:109-139](../../src/rurixc/src/dxil_spirv.rs))**无** `OpImageSample*` / `OpImageFetch` / `OpImageRead` / `OpImageWrite` / `OpImageQuery` 任一;`emit_resource`([src/rurixc/src/dxil_spirv.rs:494-568](../../src/rurixc/src/dxil_spirv.rs))**只** emit `OpTypeImage`/`OpTypeSampler` + `UniformConstant` 全局变量 + `DescriptorSet`/`Binding` 装饰(**opaque 绑定声明**),body 内**无法对其采样/取数**。

结论:图形=B 路**结构上无纹理访问 opcode 路径**;资源句柄仅可作为 opaque 绑定声明 emit,不可在着色 body 中读取其内容。

### 2.2 现有最复杂的 Rurix 图形着色器语料不涉采样

`conformance/dxil/graphics/accept/` 现有语料:`vs_varying.rx` / `vs_body_output.rx` / `fs_passthrough.rx` / `fs_body_arith.rx` / `vs_fs_link.rx`。其中 `fs_body_arith.rx` 注释逐字写明"**不涉及纹理/采样/资源访问语义**"——这是当前 Rurix 图形源能表达的最复杂 body(读输入 varying + f32 算术 + 写输出)。

`tests/dxil/dxil_golden.rs::binding_layout_digest_golden_matches` 内联语料 `fragment fn fs_main(tex: Texture2D<f32>, samp: Sampler) -> FsOut { FsOut { color: 0.0 } }` 证明:`Texture2D<f32>` / `Sampler` 作为**资源形参类型**可被前端解析 + 资源绑定装饰可 emit,但 body `FsOut { color: 0.0 }` **从不读取 `tex`/`samp`**——即语言当前**无读取资源内容的语法**,资源句柄仅止于声明 + 绑定。

### 2.3 uc04-demo device 段恒 blocked

`src/uc04-demo/src/device.rs::execute_offscreen`([src/uc04-demo/src/device.rs:36-46](../../src/uc04-demo/src/device.rs))恒返回 `Uc04Error::BlockedOnRd013`,纯 host/safe、零 D3D12 FFI、零 `windows` crate 依赖;host 侧 RXS-0167~0170 装配/编排模型已全落(复用 RFC-0005 RTS0,P-11)。

### 2.4 RD-021 / 06§4.2 禁区

`registry/deferred.json` RD-021 `reason` 逐字:纹理路径内存模型映射(采样/load/store opcode、缓存一致性、LOD/导数、越界采样后果、memory-order)属 06§4.2 🔒 禁区 + 硬规则 5,须agent 经 Full RFC 落笔,AI 不发明采样语义本体;触及即停手。`evidence/uc04_pr_f2_preflight_blockers_20260628.md` 第 3 条已记录 RD-021 为第二边界。

## 3. 决策树判定

UC-04 deferred 的本质 = 几何 pass 写 G-buffer(MRT)→ **lighting pass 读 G-buffer(SRV 采样)** 做延迟着色 → offscreen readback 对照。其中"lighting pass 读 G-buffer"是唯一卡点:

- 着色器读一张 SRV 纹理的内容,在 DXIL/SPIR-V 里**必须**有纹理访问 opcode(`Sample*` 或 `Fetch/Load`)。
- 这同时撞两道墙:
  1. **RXS-0171 白名单不含资源/纹理/采样访问 → `RX6013`**(§2.1):Rurix 源**结构上无法表达** G-buffer 读取(无采样语法,`emit_spirv_body` 无采样 opcode 路径)。
  2. **RD-021 / 06§4.2 纹理路径内存模型映射禁区**(§2.4):采样 opcode/LOD/导数/越界/缓存一致性/memory-order 须 agent Full RFC 落笔,硬规则 5 + 本任务铁律明令**不得顺手发明采样语义本体强行出绿**。

**第一分支(采样限定在 opaque 句柄 + D3D12 视图绑定层、不由 Rurix source 定义采样 opcode)不成立**:当前 Rurix 源无任何采样/取数语法,`emit_spirv_body` 无 `OpImageSample*`/`Fetch` 路径,资源句柄仅止于声明 + 绑定,**不存在**"lighting pass 读 G-buffer 但不触采样语义本体"的合法机制。强行出绿只能靠手写 HLSL/外部 DXIL/固定像素/单 pass/host-only——全被 G-G2-4 防降级硬门 + 本任务"非选项"清单禁止。

**判定:停手分支。** 本任务不出 device 绿;`execute_offscreen` 维持 `BlockedOnRd013` 不解;G-G2-4 维持 blocked-honest。
