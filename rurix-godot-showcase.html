Rurix-Godot：安全可证的 Godot GPU 加速桥

为 Godot 4.7-dev Windows D3D12 Forward+ 渲染管线提供可选式、可回退、可审计的 GPU pass 替换方案， 在不破坏引擎原渲染路径的前提下，用 Rurix 静态编译语言释放 NVIDIA GPU 的底层并发潜力。

硬件交互 Godot 引擎优化 D3D12 Forward+ GPU 系统编程

1创意介绍

开源游戏引擎 Godot 在 Windows D3D12 Forward+ 管线下，部分 GPU pass 仍采用通用实现，难以充分压榨 NVIDIA GPU 的并行能力； 而开发者若直接用 CUDA/HLSL 改写，又容易在资源生命周期、并发时序与内存安全上“踩坑”。

痛点

Godot 默认 pass 性能天花板明显；深度魔改渲染器维护成本高，且容易引入内存/并发 bug。

灵感

Rurix 已把资源所有权、流序分配、CUDA/D3D12 互操作做成类型系统一等公民，可像“外接加速卡”一样桥接 Godot。

产品形态

Godot opt-in 模块 modules/rurix_accel + Rurix C ABI 桥接库 rurix_godot.dll。

2目标用户与使用场景

独立开发者 & 小型工作室

用 Godot 开发 Windows PC 3D 游戏，希望在不重写渲染器的情况下获得更稳定的帧率。

引擎技术团队

需要深入研究 Forward+ 渲染瓶颈，验证新的 GPU pass 实现与视觉一致性。

图形程序员

希望用一门带所有权与类型安全的语言编写 GPU kernel，降低 CUDA/HLSL 的手动心智负担。

3价值与意义

效率提升

目标在 7 个标准场景下实现几何平均 FPS ≥ 1.5×、P95 帧时间降低 ≥ 30%，单场景 FPS 不跌破原版的 95%。

安全可证

Rurix 的类型系统把资源生命周期、流序分配、跨线程转移错误拦截在编译期，降低运行时崩溃与内存泄漏风险。

可审计的开源范式

Apache-2.0/MIT 双许可；所有性能声明基于 measured_local evidence JSON 与视觉 diff，推动开源引擎形成可复现优化标准。

4三阶段加速路线

每个 pass 单独启用/禁用，失败自动 fallback 回 Godot 原路径；所有阶段必须有视觉 diff 与性能证据才能推进。

阶段覆盖 Pass核心收益状态
Tier 1 · 低风险 luminance reduction、tonemap、SSAO/SSIL blur、TAA resolve、particles copy 快速验证 ABI 与视觉一致性，小幅降低 dispatch 开销 开发中
Tier 2 · 高收益 clustered light binning、GPU culling、visible instance compaction、material sorting、indirect draw args 针对 Forward+ 核心瓶颈，目标贡献主要 FPS 提升 待 Tier 1 完成后启动
Tier 3 · 结构性优化 post FX fusion、descriptor/root signature cache、PSO prewarm、bindless/resource-array 减少 barrier、PSO churn 与启动卡顿，进一步压低 P95 规划中

5证据链与验收门

性能声明必须有 measured_local evidence，禁止 estimated 占位；每道 CI 门都经过“构造缺陷 → 红 → 复原 → 绿”的真实红绿验证。

7 场景基准

clustered_lights、many_mesh_instances、material_variants、post_fx_chain、volumetric_fog、particles、mixed_forward_plus；warmup 300、sample 2000、vsync off、1920×1080。

视觉 diff

LDR 绝对 diff；HDR/temporal pass 使用 SSIM/PSNR + temporal stability，确保替换 pass 不损失画质。

fallback telemetry

任意 pass 编译失败、验证失败、设备不支持或视觉 diff 超阈值时，自动 fallback 并记录原因。

让 Godot 的 GPU 优化可验证、可回退、可复用

Rurix-Godot 不是“重写渲染器”，而是为开源游戏引擎提供一张安全可控的 GPU 加速升级卡。 从 Tier 1 低风险 pass 到 Tier 3 结构优化，每一步都用真实数据说话。
