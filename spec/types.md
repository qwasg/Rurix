# Rurix 语言规范 — 类型与检查语义

> 条款:RXS-0039 ~ RXS-0047(首批,M2.2)。体例见 [README.md](README.md)。
> 依据:05 §2(类型系统基调、trait 单态化子集 D-104)、05 §9(泛型/const 泛型 D-111);07 §3(HIR body 局部推断、签名全标注、单态化导向求解);11 §3(M2 host 子集:函数/struct/enum/泛型单态化雏形)。
> 本文为已选定决策的初版条款化(档位 Direct);任何偏离 05/07/13 已锁定决策的修改须按 10 §3 升档。
> **M2.2 容忍口径**(实现要求级,随 M2.3+ 收紧):名称解析容忍区的 `Err` 类型参与的一切检查静默通过(不级联报错);`for` / `?` / 闭包的类型检查容忍(desugar 与 lang-item 随 M2.3 评估);未在本批条款覆盖的构造不产生类型诊断。

### RXS-0039 原生类型与字面量默认型

**Syntax**:原生类型名(类型位置保留名,RXS-0034):`i8 i16 i32 i64 u8 u16 u32 u64 usize f32 f64 bool char str`;单元类型 `()`。

**Legality**:

- 整数字面量(RXS-0006)的类型:有后缀取后缀型;无后缀经推断定型;推断结束仍未定型时**默认 `i32`**。
- 浮点字面量(RXS-0007)同理,默认 `f64`;字符串字面量为 `&str` 形态(MVP:`str` 仅经引用出现),字符为 `char`,`true`/`false` 为 `bool`。
- 无尾表达式的块、`return;`、单元元组 `()` 的类型为 `()`。

> 锚定测试:`conformance/typeck/literals_defaults.rx`;typeck 单测。

### RXS-0040 函数签名全标注

**Legality**:

- 函数(全部着色)形参与返回类型必须完整标注;**无全程序推断**(07 §3)。形参类型语法已由 RXS-0014 强制;返回类型缺省即 `()`。
- `const` / `static` 项必须显式标注类型(RXS-0018 已在语法层强制);其初始化器按标注类型检查 → 违例 `RX2001`。

> 锚定测试:`conformance/typeck/signatures.rx`;typeck 单测。

### RXS-0041 body 内局部类型推断

**Legality**:

- 类型推断的作用域是单个 body(函数体/初始化器);跨 body 仅经签名(RXS-0040)。形态为 Hindley-Milner 风格合一,子类型仅限生命周期(M3+ 条款化)。
- `let` 绑定:有标注时初始化器与标注合一 → 违例 `RX2001`;无标注时绑定类型 = 初始化器类型;无初始化器时为推断变量,首次赋值定型。
- 赋值 `lhs = rhs`:两侧类型合一 → 违例 `RX2001`;复合赋值按对应二元运算规则(RXS-0043)。
- body 结束时仍未定型的推断变量按字面量默认化(RXS-0039)收敛;数值默认化不可达的悬空变量不报错(M2.2 容忍,M2.3 评估收紧)。

> 锚定测试:`conformance/typeck/inference_locals.rx`;typeck 单测。

### RXS-0042 调用一致性

**Legality**:

- 仅函数项(含关联函数)与 fn 指针类型可被调用;对非可调用类型施加调用 → `RX2005`。
- 实参数目必须与形参数目一致 → 违例 `RX2003`(`self` 形参不计入显式实参)。
- 每个实参与对应形参类型合一 → 违例 `RX2001`;调用表达式的类型 = 返回类型。
- `return e` / 块尾值与函数返回类型合一 → 违例 `RX2001`;`return;` 要求返回类型 `()`。

> 锚定测试:`conformance/typeck/calls.rx`;`tests/ui/typeck/`(实参数目/返回不符)。

### RXS-0043 运算符类型规则

**Legality**:

- 算术 `+ - * / %` 与位运算 `& | ^ << >>`:两操作数为**同一**数值类型(整数或浮点;位运算仅整数),结果同型 → 违例 `RX2006`。M2.2 内建直接检查,不经运算符 trait(经 trait 的可重载形态随 M2.3+ 评估,RXS-0045 口径)。
- 比较 `== != < > <= >=`:两操作数同型(数值/bool/char),结果 `bool` → 违例 `RX2006`。
- 逻辑 `&& ||`:两操作数 `bool`,结果 `bool`;`!` 对 `bool` 取非或对整数取反;一元 `-` 仅数值 → 违例 `RX2006`。
- `if` / `while` 条件与 `match` guard 必须为 `bool` → 违例 `RX2001`。
- 区间 `a..b` / `a..=b`:两端同整数型(`for` 迭代该区间时绑定为该型)。

> 锚定测试:`conformance/typeck/operators_typed.rx`;`tests/ui/typeck/`(非法运算数)。

### RXS-0044 struct / enum 构造与字段访问

**Legality**:

- struct 字面量:字段名必须属于该 struct → 违例 `RX2002`(未知字段);每个已声明字段必须恰好初始化一次 → 缺失/重复 `RX2002`;字段值与字段类型合一 → `RX2001`。
- 元组结构体/元组变体构造按调用一致性(RXS-0042)检查。
- 字段访问 `e.f`:`e` 为 struct(或其引用,自动解一层引用)且 `f` 为其字段 → 违例 `RX2002`;元组字段 `e.0` 按位置裁决。
- `match` 各臂体类型合一(整体类型 = 合一结果);`if`/`else` 两分支类型合一;无 `else` 的 `if` 为 `()` → 违例 `RX2001`。

> 锚定测试:`conformance/typeck/adt_construct.rx`;`tests/ui/typeck/`(未知/缺失字段、臂型不一)。

### RXS-0045 泛型单态化雏形

**Legality**:

- 全部泛型单态化(D-111);类型实参经 turbofish 显式给出或自实参推断;实参替换后按具体类型检查。
- 泛型 bound(RXS-0020)在 M2.2 仅记录**不求解**:bound 不满足不产生诊断(trait 求解的单态化导向简化版随 M2.3+ 条款化,本条款只追加扩展)。
- const 泛型参数(`const N: usize`)在值位置的类型 = 其标注类型;数组长度位置接受 const 参数与整数字面量。

> 锚定测试:`conformance/typeck/generics_mono.rx`;typeck 单测。

### RXS-0046 inherent 方法与 `as` 转换

**Legality**:

- 方法调用 `recv.m(args)`:在接收者类型(struct/enum,自动解一层引用)的 inherent impl 关联函数中查找 `m` → 未找到 `RX2004`;`self` 接收者形态(值/引用)在 M2.2 宽松匹配(精确借用检查属 M3)。trait 方法查找随 M2.3+(RXS-0045 口径)。
- `as` 转换:仅数值类型之间(整数↔整数、整数↔浮点、浮点↔浮点)与 `bool`/`char` → 整数 → 违例 `RX2001`;结果类型 = 目标类型。

> 锚定测试:`conformance/typeck/methods_casts.rx`;`tests/ui/typeck/`(未知方法)。

### RXS-0047 类型检查诊断要求

**Legality**:类型检查阶段的首批结构化错误码:

- `RX2001`: 类型不匹配(合一失败/条件非 bool/转换非法/返回不符)
- `RX2002`: 未知或缺失字段
- `RX2003`: 实参数目不符
- `RX2004`: 无此方法或关联项
- `RX2005`: 不可调用类型
- `RX2006`: 非法运算数

**Implementation Requirements**:

- `RX2001` 诊断应同时给出期待类型与实际类型的渲染文本。
- `RX2003` 应给出期待/实际实参数目;`RX2002`/`RX2004` 应携带字段/方法文本名。
- **Err 容忍不级联**:任一操作数/字段/接收者类型为容忍区 `Err` 时,相关检查静默通过,不得产生派生诊断(防一错多报;M2.1 类型位置容忍口径的延续)。
- 诊断 span 指向违例表达式本体(而非其子树)。

> 锚定测试:`tests/ui/typeck/` 黄金路径 2 snapshot 集;typeck 单测。
