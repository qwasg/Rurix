# Rurix Tutorial

[English](README.en.md) · [简体中文](README.md)

> **API-convergence notice (RD-008):** the Rurix MVP is complete, but the public surface is still converging and the stable-API snapshot is not yet frozen. The syntax and library names in this tutorial **may change between versions**. Every independently compilable example is held under a CI gate (`ci/tutorial_smoke.py`), so any change to the language surface trips an alarm — the tutorial code therefore always matches the current toolchain.

This is a progressive, hands-on tutorial for **writing your first Rurix program from scratch**. It is aimed at developers who already understand Rust and the basic concepts of GPU programming and want to get going with Rurix. Unlike the **reference documentation site** generated by `rx doc` (spec clauses / error codes / traceability matrix), this tutorial is a **narrative learning path**.

## Before you start

- Environment: Windows 11 + an NVIDIA GPU, the CUDA Toolkit, and MSVC 2022 (see [`00_install.en.md`](00_install.en.md)).
- You can run `cargo build --workspace` successfully at the repository root.
- Want to read real, compilable code directly? See [`../conformance/tutorial/`](../conformance/tutorial/) — the single source of truth for every independently compilable example in this tutorial.

## Compilable examples vs. reference snippets

The code in this tutorial comes in two kinds — keep the distinction in mind:

- **Compilable examples** (`conformance/tutorial/*.rx`): only the language surface that `rx check` can parse standalone (host / `device fn` / `const fn` / kernel definitions), run end-to-end by the CI gates `rx check` + `rx run`. You can copy them verbatim and they will compile.
- **Reference snippets**: launch (`stream.launch`), runtime resources (`Context` / `Stream` / `Buffer`), stdlib math (`Vec3` / `Mat4`), etc. require a **package context and a runtime** to resolve. The tutorial presents these as snippets and links them to existing conformance corpora (already constrained by the parse gate) and to the real demos under `src/uc02-demo` and `src/uc03-demo`.
